### A.2. 技術仕様回答（入札回答用）

```
【技術仕様回答】

Q1: 異なるネットワーク間のファイル転送時に無害化処理を行うか？
A1: 是。PdfStreamParser（PDF）、OoxmlStreamParser（Office）、ZipSanitizer
    （ZIP）、SvgSanitizer（SVG）の各形式について True CDR 処理を実施します。
    また、日本固有の JTD 形式については PDF 変換により無害化を行います。
    ※ DS-200 セキュリティ・バイ・デザインガイドライン第3章に準拠

Q2: DS-100 が求める監査証跡（Audit Trail）は可能か？
A2: 可能。JSON / Syslog / CEF の各形式で構造化ログを出力します。
    各イベントには UUID、ISO 8601 タイムスタンプ（UTC）、SHA-256 
    ハッシュ値、ユーザー識別子、処理結果を含みます。
    ※ DS-100 Normative 項目「ログ記録・監査証跡」に完全対応

Q3: マイナンバー等の個人情報を検出できるか？
A3: 可能。RegexPIIDetector により、マイナンバー（12 桁）、クレジットカード番号、
    メールアドレス、電話番号等を自動検出します。
    Block / Mask / AlertOnly のアクションを選択可能です。
    ※ マイナンバー法および APPI に準拠

Q4: 一太郎（.jtd）形式に対応しているか？
A4: 対応。LibreOffice または一太郎ビューア経由で PDF に変換後、
    既存 CDR パイプラインに投入して無害化処理を行います。
    ※ 本機能は競合製品（VOTIRO、FileZen）では非対応です。

Q5: Active Directory との連携は可能か？
A5: 可能。LDAP/LDAPS (port 389/636) による Bind 認証、
    グループベース認可、OU 単位検索に対応します。
    ※ DS-200 セキュリティ・バイ・デザインガイドラインに準拠

Q6: TLS による通信暗号化は実装されているか？
A6: 是。gRPC および HTTP 通信の全経路で TLS 1.2+ を使用します。
    証明書ピニングにも対応しています。
    ※ DS-100 Normative 項目「通信経路の暗号化」に完全対応

Q7: 和暦日付に対応しているか？
A7: 対応。JapaneseCalendarProvider により令和/平成/昭和等の元号を含む
    日付の正規化・変換を行います。
    ※ 本機能は競合製品では非対応です。
```
